## 目标
- 完成店铺详情页内的“菜单数字化”流程：录入菜品（名称、价格）、查看已录入列表、点赞与报错交互。
- 对齐现有后端接口（`/api/menus`、`/api/menus/:id/like`、`/api/menus/:id/report`），修正前端 API 与页面实现。

## 前端实现
### API 对齐
- 新增/修正前端 `api/menu.js`：
  - `getMenuItems(storeId, { page, limit })` → `GET /api/menus?storeId=xxx`
  - `digitizeMenu(payload)` → `POST /api/menus`（payload：`{ storeId, name, price }`）
  - `likeMenuItem(id)` → `POST /api/menus/:id/like`
  - `reportMenuItem(id, reason)` → `POST /api/menus/:id/report`
- 统一使用请求拦截器（已接入），自动附带 Token。

### 页面与交互
- `pages/menu/digitize.vue`
  - 上方：引用店铺“原始菜单图”缩略图预览（来自 Store 详情的 `menuImages`）。
  - 中部：录入表单（菜名、价格），价格校验（>0，最多两位小数），提交成功后清空并刷新列表。
  - 下方：已录入列表（分页），每项显示：名称、价格、点赞数、报错数；操作按钮：点赞、报错。
  - 空态与错误态提示；重复提交保护；loading 状态。
- `pages/menu/item-detail.vue`
  - 展示单项信息与互动统计；提供点赞/报错；从列表可跳转进来。

### 体验优化
- 表单校验与错误提示：必填、类型范围；接口失败提示。
- 列表分页与下拉刷新：首屏 20 条；滚动到底自动加载；下拉刷新重载。
- 交互反馈：点赞/报错本地计数即时+1；失败回滚。

## 后端对齐（如需）
- 已有：`POST /api/menus`、`GET /api/menus?storeId`、`POST /api/menus/:id/like`、`POST /api/menus/:id/report`。

  - 录入时校验 price ≥ 0；`creator` 使用 `req.userId`。

## 验证
- 用一个店铺（包含 `menuImages`）进入“菜单数字化”页：录入 3–5 个菜品，价格格式合规。
- 点赞同一菜品多次不会重复计数；取消点赞按后端实现逻辑展示结果。
- 报错多次达到阈值，菜品 `status` 改为 `pending` 时前端有明显标识。
- 列表分页、刷新与错误提示正常。
